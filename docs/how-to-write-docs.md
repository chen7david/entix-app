# How to Write Documentation

Follow these rules to keep our docs useful and clear.

## Style Rules
1. **Simple Language**: Use plain English and short sentences.
2. **Step-by-Step**: Use numbered lists (1., 2., 3.) for all procedures.
3. **One Idea per Section**: Do not cram multiple concepts into one header.
4. **No Inline "Why"**: Put rationale in separate "Why this way?" sections or files. Link to them.
5. **Real Examples**: Use tiny, copy-paste-ready code from this app.
6. **Zero Fluff**: No jargon without immediate explanation. No repetitive filler.

## Structure
- Start with a clear objective.
- Provide the "How-to" steps.
- Provide a "Verify it works" section.
- End with the timestamp and a link to this guide.

## The 5-Minute Test
Before publishing, ask: **"Is this clear to a new dev in 5 minutes?"**
